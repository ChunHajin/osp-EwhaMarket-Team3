# osp-EwhaMarket-Team3
오픈SW플랫폼 3조 EwhaMarket 프로젝트

## 프로젝트 개요

이화마켓은 대학교 학생들을 위한 중고 물품 거래 웹 플랫폼입니다. Flask를 기반으로 개발되었으며, 사용자 간 안전하고 편리한 물품 등록, 조회, 거래를 지원하는 오픈 소스 프로젝트입니다.

-----

## 주요 기능

본 프로젝트는 다음과 같은 핵심 기능들을 제공합니다.

| 분류 | 기능 상세 |
| :--- | :--- |
| **사용자 관리** | 회원가입 시 비밀번호를 **SHA-256 해시 형태로 저장**하여 보안을 강화합니다. 로그인/로그아웃 및 안전한 세션 관리를 지원합니다. |
| **상품 거래** | 상품 등록, 조회, 수정 및 삭제(CRUD)가 가능합니다. 상품 상태를 '거래 완료'로 변경하여 구매자를 지정할 수 있습니다. |
| **개인화 서비스** | **찜하기 기능**을 통해 상품에 대한 좋아요 상태를 토글하고 실시간으로 찜 개수를 반영합니다. |
| **마이페이지 특화** | 판매 중/완료 상품 수를 명확히 표시하며, 프로필 사진을 즉시 변경할 수 있습니다. 구매 내역에서 리뷰 작성 여부에 따라 **'리뷰 작성' 또는 '리뷰 보기' 링크를 동적으로 제공**합니다. |
| **리뷰 기능** | 구매 완료된 상품에 대해 리뷰 내용, 별점, 작성 시간 등을 저장하여 리뷰를 작성할 수 있습니다. 찜 목록에서는 채워진 하트 버튼을 클릭하여 **경고창 없이 즉시 찜을 해제**할 수 있습니다. |

## 기술 스택

| 분류 | 기술 | 상세 내용 |
| :--- | :--- | :--- |
| **Backend** | Python (Flask) | 웹 애플리케이션 프레임워크로 사용되었습니다. |
| **Database** | Firebase Realtime Database | 데이터 저장 및 실시간 관리에 사용됩니다. |
| **DB Wrapper** | Pyrebase | Python에서 Firebase 접근을 지원합니다. |
| **Frontend** | HTML5, CSS3, JavaScript | Jinja2 템플릿을 사용하여 인터페이스를 구성하고 동적 기능을 구현했습니다. |

## 코드 정리 및 문서화 수준

오픈 소스 프로젝트의 표준을 따르기 위해 다음 사항들을 보강했습니다.

  * **Docstring 및 타입 힌트 (Type Hint) 적용:** `database.py`와 `app.py`의 모든 함수에 Docstring을 활용하여 함수 설명, 매개변수, 반환 값을 명시했으며, 파이썬 타입 힌트를 적극적으로 사용하여 데이터 타입을 명확히 했습니다.
  * **핵심 로직 인라인 주석:** DB 접근, 비밀번호 해시화, 세션 관리 등 핵심 로직 부분에 구현 목표와 작동 방식을 설명하는 인라인 주석을 추가했습니다.
  * **설정 관리의 명확화:** `app.py`에서 `SECRET_KEY` 등 중요한 설정 값들을 **환경 변수**를 통해 로드하도록 하여, 소스 코드 공개 시 민감 정보 노출을 방지하고 오픈 소스 배포 용이성을 확보했습니다.

## 설치 및 실행 방법

### 1\. 환경 설정 및 종속성 설치 (Conda 환경 기반)

본 프로젝트는 **Anaconda** 환경에서 개발되었으며, Conda 가상 환경을 기반으로 실행하는 것을 권장합니다.

1.  **Anaconda 및 VS Code 설치:** Anaconda와 Visual Studio Code가 설치되어 있어야 합니다.
2.  **Conda 환경 활성화:** 
    ```bash
    # 가상환경 생성
    # conda create -n osp_env python=<python 버전>
    ```
    ```bash
    # 생성한 가상환경 활성화
    # conda activate osp_env
    ```
    ```bash
    # 플라스크 설치
    # conda install flask
    ```

3.  **VS Code Interpreter 설정:** VS Code를 실행한 후, `Python: Select Interpreter` 명령을 통해 `osp_env` 환경의 Python 인터프리터를 선택합니다.

### 2\. Firebase 설정

1.  **Firebase 프로젝트 구성:** Google Firebase에서 프로젝트를 생성하고 Realtime Database를 활성화합니다.
2.  **서비스 계정 키 파일:** 프로젝트 설정에서 생성한 서비스 계정 키 파일(`firebase_auth.json` 등)을 다운로드하여 `backend/authentication/` 경로에 저장해야 합니다.
3.  **환경 변수 설정:**
      * `SECRET_KEY`: Flask 세션 관리를 위한 비밀 키를 설정합니다.
      * `FIREBASE_CONFIG_PATH`: 서비스 계정 키 파일의 경로를 설정합니다 (기본값: `backend/authentication/firebase_auth.json`).

### 3\. 애플리케이션 실행

프로젝트 폴더를 VS Code에서 연 상태에서 다음 절차를 따릅니다.

1.  **터미널 열기:** VS Code의 **Terminal** 메뉴에서 **New Terminal**을 선택하고, **Command Prompt**를 선택하여 터미널을 엽니다. (이때 가상 환경 `(osp_env)`가 활성화되어 있는지 확인합니다.)
2.  **Flask 서버 실행:** 다음 명령어를 입력하여 서버를 시작합니다.
    ```bash
    python backend/app.py
    flask --app backend/app.py --debug run
    ```
3.  **접속:** 서버가 실행되면 웹 브라우저에서 `http://127.0.0.1:5000/` 주소로 접속하여 서비스를 이용할 수 있습니다.
